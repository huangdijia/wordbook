# Spec 拆分计划：纯静态英文单词本

## 1. 拆分目标

根据 `docs/prd.md`，将 V1.0「基础单词卡片版」拆分为可独立交付、可验证的 spec。每个 spec 都围绕一个明确业务目标展开，避免把后续版本能力提前做进第一版。

## 2. 基本假设

1. 第一版只做纯静态网页，不引入后端、数据库、构建工具或登录体系。
2. 数据源固定为 `data/config.json` 和 `data/words/*.json`。
3. 页面通过本地静态服务或静态托管访问，不保证 `file://` 方式可用。
4. V1.0 不实现 `localStorage` 学习记录、掌握标记、发音、拼写测试、批量导入和后台管理。

## 3. Spec 总览

| Spec | 名称 | 目标 | 优先级 | 依赖 |
|---|---|---|---|---|
| S01 | 项目静态骨架 | 建立可直接打开的静态页面结构 | P0 | 无 |
| S02 | 数据文件与加载 | 定义并读取配置和词库 JSON | P0 | S01 |
| S03 | 年级 / 单元 / 模式选择 | 让用户选择练习范围和练习模式 | P0 | S02 |
| S04 | 练习列表生成 | 按顺序或随机模式生成当前单元单词列表 | P0 | S03 |
| S05 | 单词卡片与翻转 | 默认显示中文，点击翻转显示英文 | P0 | S04 |
| S06 | 上一个 / 下一个与进度 | 支持单词切换和进度展示 | P0 | S05 |
| S07 | 空状态与加载失败提示 | 对无词库、加载失败等情况给出清晰反馈 | P1 | S02-S06 |
| S08 | 响应式与验收检查 | 保证手机、平板、桌面可用并完成 PRD 验收 | P0 | S01-S07 |

## 4. Spec 详情

### S01：项目静态骨架

**目标**

建立 PRD 要求的纯静态目录结构，并让页面具备基础容器。

**范围**

- 新增 `index.html`。
- 新增 `assets/css/style.css`。
- 新增 `assets/js/app.js`。
- 新增 `data/` 目录占位。
- 页面包含标题区、筛选区、卡片区、操作区、提示区的基本 DOM 节点。

**不做**

- 不引入框架或打包工具。
- 不实现业务交互。

**交付物**

- `index.html`
- `assets/css/style.css`
- `assets/js/app.js`

**验收**

- 打开页面后可以看到页面基础结构。
- 对应 PRD 验收项：A001 的静态结构部分。

---

### S02：数据文件与加载

**目标**

按照 PRD 定义拆分配置和词库，并在前端读取。

**范围**

- 新增 `data/config.json`。
- 新增 `data/words/` 下的按册次词库 JSON。
- 在 `assets/js/app.js` 中通过 `fetch` 读取两个 JSON 文件。
- 维护基础运行状态：`config`、`words`、`loading`、`error`。
- 加载成功后渲染页面标题、副标题。

**不做**

- 不引入后端、数据库或构建工具。
- 不做复杂 schema 校验，只做必要字段读取和错误提示。

**交付物**

- `data/config.json`
- `data/words/*.json`
- JSON 加载逻辑

**验收**

- `config.json` 和按册次词库 JSON 职责分离。
- 页面标题、副标题来自 `config.json`。
- 词库数据来自当前册次的词库 JSON。
- 加载失败时能进入错误提示流程。
- 对应 PRD 验收项：A002、A003、D001。

---

### S03：年级 / 单元 / 模式选择

**目标**

让用户基于配置选择年级、单元和练习模式。

**范围**

- 年级选项来自 `config.grades`。
- 默认年级来自 `config.app.defaultGrade`，不存在时回退到第一个年级。
- 单元选项来自当前年级的 `units`。
- 切换年级时自动选中该年级第一个单元。
- 模式选项来自 `config.modes`。
- 默认模式来自 `config.app.defaultMode`，不存在时回退到第一个模式或 `sequence`。
- 切换年级、单元、模式时重置当前索引和卡片翻转状态。

**不做**

- 不持久化用户选择。
- 不支持跨年级或跨单元混合练习。

**交付物**

- 年级下拉框
- 单元下拉框
- 模式下拉框
- 选择状态更新逻辑

**验收**

- 切换年级后，单元列表同步更新。
- 切换年级后，默认进入该年级第一个单元。
- 切换单元后，只显示该单元单词。
- 切换模式后，练习列表重新生成。
- 对应 PRD 验收项：A009、A010。

---

### S04：练习列表生成

**目标**

根据当前年级、单元和模式生成实际练习单词列表。

**范围**

- 实现 `getWordsByUnit(words, unitKey)`。
- 实现 `shuffleWords(words)`。
- 实现 `buildPracticeWords(words, unitKey, mode)`。
- 顺序模式按 `sort` 升序展示。
- 随机模式只打乱当前单元内的单词。
- 单词字段缺失时不阻断页面，优先跳过异常单词并在控制台提示。

**不做**

- 不实现随机种子。
- 不保证每次随机都和上一次完全不同。

**交付物**

- 当前练习列表生成逻辑
- 当前单元词库筛选逻辑

**验收**

- 顺序模式按 `sort` 升序展示。
- 随机模式不跨单元。
- 当前单元无单词时返回空列表。
- 对应 PRD 验收项：A011、A012、D002、D003、D004。

---

### S05：单词卡片与翻转

**目标**

实现背单词核心体验：默认看中文，点击后查看英文。

**范围**

- 卡片默认展示当前单词的 `chinese`。
- 点击卡片后展示当前单词的 `english`。
- 再次点击卡片恢复中文。
- 英文使用更大字号。
- 卡片内容居中展示。
- 切换单词、单元、年级、模式后恢复中文正面。

**不做**

- 不播放发音。
- 不做拼写输入。
- 不做掌握 / 不熟标记。

**交付物**

- 单词卡片渲染
- 翻转状态逻辑
- 卡片视觉样式

**验收**

- 初始显示中文。
- 点击显示英文。
- 再次点击恢复中文。
- 切换范围或单词后恢复中文。
- 对应 PRD 验收项：A004、A005、A006。

---

### S06：上一个 / 下一个与进度

**目标**

支持在当前练习列表内切换单词，并展示学习进度。

**范围**

- 点击「下一个」进入下一个单词。
- 点击「上一个」返回上一个单词。
- 第一个单词时禁用「上一个」。
- 最后一个单词时禁用「下一个」或展示完成态。
- 进度格式为 `当前序号 / 当前单元总单词数`。
- 切换单词后恢复中文正面。

**不做**

- 不自动跳转下一单元。
- 不记录已学进度。

**交付物**

- 上一个按钮
- 下一个按钮
- 进度展示
- 单词切换逻辑

**验收**

- 上一个 / 下一个边界状态正确。
- 进度展示正确。
- 切换单词后卡片恢复中文。
- 对应 PRD 验收项：A007、A008、A013。

---

### S07：空状态与加载失败提示

**目标**

保证异常或无数据时页面不空白、不崩溃。

**范围**

- 当前单元无单词时显示：`当前单元暂无单词，请检查当前册词库配置。`
- `config.json` 加载失败时显示：`配置文件加载失败，请检查 data/config.json`。
- 当前册词库加载失败时显示具体文件路径，例如：`词库文件加载失败，请检查 data/words/grade_1a.json`。
- JSON 格式错误时归入对应加载失败提示。
- 空状态下仍保留年级、单元、模式选择区域。

**不做**

- 不做在线编辑或修复 JSON。
- 不做复杂错误详情展示给儿童用户。

**交付物**

- 空状态组件或提示区域
- 加载失败提示逻辑

**验收**

- 当前单元无单词时不显示空白卡片。
- 加载失败时有明确提示。
- 筛选区域在空单元场景仍可使用。
- 对应 PRD 验收项：A014。

---

### S08：响应式与验收检查

**目标**

完成第一版整体视觉可用性和验收回归。

**范围**

- 页面最大宽度控制在 480px - 720px。
- 手机单列布局。
- 按钮高度不小于 44px。
- 卡片高度适合儿童点击和阅读。
- 中文、英文居中且字号符合 PRD 建议范围。
- 使用本地静态服务验证页面。

**不做**

- 不做多主题皮肤。
- 不引入复杂动画，只保留必要的卡片翻转反馈。

**交付物**

- 响应式样式
- 验收检查记录

**验收**

- 手机、平板、桌面宽度下页面可正常使用。
- 通过 PRD A001-A014 和 D001-D005 的人工验收。
- 本地预览命令可用：

```bash
python3 -m http.server 8080
```

## 5. 建议实施顺序

```text
1. S01 项目静态骨架
   验证：页面基础结构可见。

2. S02 数据文件与加载
   验证：标题、副标题、配置和词库能从 JSON 读取。

3. S03 年级 / 单元 / 模式选择
   验证：三个选择器能正确联动并重置状态。

4. S04 练习列表生成
   验证：顺序 / 随机模式按规则生成当前单元列表。

5. S05 单词卡片与翻转
   验证：中文和英文可点击切换。

6. S06 上一个 / 下一个与进度
   验证：边界禁用、进度、切换重置均正确。

7. S07 空状态与加载失败提示
   验证：无词和加载失败都有清晰提示。

8. S08 响应式与验收检查
   验证：完成 PRD 验收项 A001-A014、D001-D005。
```

## 7. MVP 边界

本计划只覆盖 PRD V1.0。以下内容即使实现成本不高，也不在本轮 spec 中实现：

- 用户登录、后台管理、云端同步。
- `localStorage` 学习记录。
- 掌握 / 不熟标记。
- 只复习不熟单词。
- 自动发音。
- 拼写测试。
- Excel / CSV 批量导入。
- 多主题皮肤。
