# PRD：纯静态英文单词本

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 产品名称 | 英文单词本 |
| 产品形态 | 纯静态网页 |
| 目标用户 | 小学生 / 家长 |
| 核心场景 | 通过卡片翻转方式背英文单词 |
| 数据方式 | JSON 配置 |
| 是否需要后端 | 否 |
| 是否需要数据库 | 否 |

---

## 2. 产品背景

为了帮助孩子更轻松地背诵英文单词，需要设计一个简单、直观、低成本维护的英文单词本工具。

第一版以「纯静态页面 + JSON 配置」方式实现，家长或维护者只需要修改 JSON 文件即可更新年级、单元和词库内容。孩子进入页面后，可以选择年级和单元，通过卡片翻转方式进行单词记忆。

---

## 3. 产品目标

### 3.1 核心目标

1. 支持按「年级 / 单元 / 单词」组织词库。
2. 支持中文卡片展示，点击后翻转显示英文。
3. 支持顺序模式和随机模式。
4. 使用 JSON 文件维护词库和页面配置。
5. 无需后端、无需数据库，可直接静态部署。

### 3.2 非目标

第一版暂不支持以下能力：

1. 用户登录。
2. 后台管理系统。
3. 云端同步。
4. 多用户学习记录。
5. 复杂学习统计。
6. 自动发音。
7. 拼写测试。

---

## 4. 用户角色

| 角色 | 说明 | 主要行为 |
|---|---|---|
| 家长 / 维护者 | 负责维护词库和配置 | 修改 JSON 文件，更新年级、单元、单词 |
| 孩子 / 学习者 | 使用页面背单词 | 选择年级和单元，翻卡片背单词 |

---

## 5. 使用场景

### 5.1 场景一：按教材单元背单词

孩子选择对应年级和单元，系统按照配置顺序展示单词。孩子看到中文后，先自己回忆英文，再点击卡片查看答案。

### 5.2 场景二：随机抽背复习

孩子已经学过某个单元，希望检查是否真正掌握。切换到随机模式后，系统打乱单词顺序，进行抽背练习。

### 5.3 场景三：家长更新词库

家长根据学校教材内容，直接修改 `data/words/` 下对应册次的词库 JSON，新增或调整单词。页面无需重新开发，刷新后即可使用新词库。

## 7. 功能需求

## 7.1 配置管理

### 功能说明

系统通过 `config.json` 管理页面基础配置、年级、单元和练习模式。

### 配置内容

| 配置项 | 说明 | 示例 |
|---|---|---|
| app.title | 页面标题 | 英文单词本 |
| app.subtitle | 页面副标题 | 每天背一点，英语进步一点 |
| app.defaultGrade | 默认年级 | grade3 |
| app.defaultMode | 默认模式 | sequence |
| modes | 练习模式配置 | 顺序模式、随机模式 |
| grades | 年级与单元配置 | 三年级、四年级 |

### 示例配置

```json
{
  "app": {
    "title": "英文单词本",
    "subtitle": "每天背一点，英语进步一点",
    "defaultGrade": "grade3",
    "defaultMode": "sequence"
  },
  "modes": [
    {
      "key": "sequence",
      "name": "顺序模式"
    },
    {
      "key": "random",
      "name": "随机模式"
    }
  ],
  "grades": [
    {
      "key": "grade3",
      "name": "三年级",
      "units": [
        {
          "key": "unit1",
          "name": "Unit 1",
          "title": "我的学校"
        },
        {
          "key": "unit2",
          "name": "Unit 2",
          "title": "我的家庭"
        }
      ]
    }
  ]
}
```

## 7.2 词库管理

### 功能说明

系统通过 `data/words/` 下的按册次 JSON 文件管理具体单词数据。词库文件只保存单词内容，不保存页面配置。

### 字段设计

| 字段 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| id | string | 是 | 单词唯一 ID | g3-u1-001 |
| unitKey | string | 是 | 所属单元 Key | unit1 |
| chinese | string | 是 | 中文含义 | 苹果 |
| english | string | 是 | 英文单词 | apple |
| sort | number | 是 | 顺序模式排序 | 1 |

### 示例词库

```json
[
  {
    "id": "g3-u1-001",
    "unitKey": "unit1",
    "chinese": "苹果",
    "english": "apple",
    "sort": 1
  },
  {
    "id": "g3-u1-002",
    "unitKey": "unit1",
    "chinese": "香蕉",
    "english": "banana",
    "sort": 2
  }
]
```

## 7.3 年级选择

### 功能说明

用户进入页面后，可以选择要学习的年级。

### 规则

1. 年级选项来自 `config.json` 中的 `grades`。
2. 默认选中 `app.defaultGrade` 对应的年级。
3. 切换年级后，单元列表需要同步更新。
4. 切换年级后，默认选中该年级下的第一个单元。
5. 切换年级后，当前卡片重置为第一个单词。

### 展示示例

```text
年级：[ 三年级 ▼ ]
```

---

## 7.4 单元选择

### 功能说明

用户选择年级后，可以继续选择该年级下的单元。

### 规则

1. 单元选项来自当前年级的 `units`。
2. 切换单元后，在当前册词库中根据 `unitKey` 筛选单词。
3. 切换单元后，当前卡片重置为第一个单词。
4. 如果当前单元没有单词，需要显示空状态。

### 展示示例

```text
单元：[ Unit 1 我的学校 ▼ ]
```

---

## 7.5 模式选择

### 功能说明

系统支持两种背诵模式：

1. 顺序模式。
2. 随机模式。

### 顺序模式

按照当前册词库中的 `sort` 字段升序展示。

示例：

```text
apple → banana → cat → dog
```

### 随机模式

进入练习或切换模式时，将当前单元的单词随机打乱。

示例：

```text
dog → apple → cat → banana
```

### 规则

1. 默认模式来自 `config.json` 的 `app.defaultMode`。
2. 模式选项来自 `config.json` 的 `modes`。
3. 切换模式后，当前单词列表重新生成。
4. 切换模式后，当前卡片重置为第一个单词。
5. 随机模式只打乱当前单元内的单词，不跨单元。

### 展示示例

```text
模式：[ 顺序模式 ▼ ]
```

## 7.6 单词卡片

### 功能说明

单词卡片是背单词页面的核心区域。

默认展示中文，点击卡片后翻转显示英文。再次点击卡片，可以返回中文。

### 正面

```text
苹果

点击查看英文
```

### 反面

```text
apple

点击返回中文
```

### 规则

1. 默认显示中文。
2. 点击卡片后显示英文。
3. 再次点击卡片后显示中文。
4. 切换上一个 / 下一个单词时，卡片恢复为中文正面。
5. 卡片内容居中展示。
6. 英文建议使用更大的字号，方便孩子识别。

---

## 7.7 上一个 / 下一个

### 功能说明

用户可以通过按钮切换当前单词。

### 规则

1. 点击「下一个」进入下一个单词。
2. 点击「上一个」返回上一个单词。
3. 当前是第一个单词时，「上一个」不可用。
4. 当前是最后一个单词时，「下一个」不可用，或显示「已完成」。
5. 切换单词后，卡片恢复到中文正面。
6. 页面显示当前进度。

### 展示示例

```text
[ 上一个 ]     3 / 12     [ 下一个 ]
```

---

## 7.8 进度显示

### 功能说明

显示当前背诵进度，帮助孩子知道当前学到第几个单词。

### 展示格式

```text
当前序号 / 当前单元总单词数
```

### 示例

```text
3 / 12
```

## 7.9 空状态

### 功能说明

当某个年级或单元下没有配置单词时，需要给出清晰提示。

### 展示文案

```text
当前单元暂无单词，请检查当前册词库配置。
```

### 规则

1. 不显示空白卡片。
2. 不显示错误页面。
3. 页面仍保留年级、单元、模式选择区域。

## 10.2 config.json 职责

`config.json` 负责：

1. 页面标题。
2. 页面副标题。
3. 默认年级。
4. 默认模式。
5. 支持的练习模式。
6. 年级列表。
7. 每个年级下的单元列表。

`config.json` 不负责保存具体单词。

---

## 10.3 按册次词库职责

`data/words/` 下的按册次词库文件负责：

1. 单词唯一 ID。
2. 所属单元。
3. 中文含义。
4. 英文单词。
5. 展示排序。

按册次词库文件不负责保存页面标题、模式配置、年级名称、册别名称、单元名称。

## 12. 核心逻辑

## 12.1 筛选当前单元单词

```js
function getWordsByUnit(words, unitKey) {
  return words
    .filter(word => word.unitKey === unitKey)
    .sort((a, b) => a.sort - b.sort)
}
```

---

## 12.2 随机打乱单词

```js
function shuffleWords(words) {
  return [...words].sort(() => Math.random() - 0.5)
}
```

---

## 12.3 生成练习列表

```js
function buildPracticeWords(words, unitKey, mode) {
  const selectedWords = getWordsByUnit(words, unitKey)

  if (mode === 'random') {
    return shuffleWords(selectedWords)
  }

  return selectedWords
}
```

---

## 12.4 切换单词时重置卡片

```js
function goToWord(index) {
  state.currentIndex = index
  state.isFlipped = false
}
```

## 13. 异常处理

| 场景 | 处理方式 |
|---|---|
| `config.json` 加载失败 | 显示「配置文件加载失败，请检查 data/config.json」 |
| 当前册词库加载失败 | 显示「词库文件加载失败，请检查 data/words/grade_1a.json」等具体文件路径 |
| 当前年级不存在 | 默认选中第一个年级 |
| 当前单元不存在 | 默认选中当前年级第一个单元 |
| 当前单元无单词 | 显示空状态 |
| 单词字段缺失 | 跳过异常单词，或在控制台输出警告 |
| JSON 格式错误 | 显示加载失败提示 |

## 18. 后续扩展建议

### 18.1 词库拆分

当词库数量变大时，可以按年级拆分：

```text
data/
├── config.json
└── words/
    ├── grade3.json
    ├── grade4.json
    └── grade5.json
```

也可以按年级和单元拆分：

```text
data/
├── config.json
└── words/
    ├── grade3-unit1.json
    ├── grade3-unit2.json
    ├── grade4-unit1.json
    └── grade4-unit2.json
```

当前实现按册次拆分词库，例如 `grade_1a.json` 表示一年级上册，`grade_1b.json` 表示一年级下册。

## 19. 总结

英文单词本第一版采用纯静态网页实现，通过 `config.json` 管理页面配置、年级、单元和模式，通过 `data/words/` 下的按册次 JSON 管理具体词库。孩子选择年级和单元后，以中文卡片开始练习，点击卡片翻转显示英文，并支持顺序和随机两种模式。

该方案实现成本低、维护简单、部署方便，适合作为家庭自用或小规模学习工具的 MVP。
